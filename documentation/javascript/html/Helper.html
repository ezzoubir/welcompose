closePopupTrack

void closePopupTrack(elem)
    Track the close of popups with expected refresh of Media Manager.
    We need a handling if the user do not close the popups with the appropriate buttons. For example, the user could use keyboard shortcuts or close the popup by mouse click on the window interface close button (depends on operating system).
    So this function is added to the popups html body event onunload, where its expected to close and refresh the Media Manager (with closePopup()). There is a condition, that compares wether several control vars (audit, submitted) are set (respectively are bool true) or not. On the basis of the result, we invoke functions in the parent window, like revoke the lowering of opacity and fire a alert demanding to close the popup with the appropriate button or link.
    Parameters:
      audit - Defined in closePopup().
      submitted - Avoid the execute of the onunload event.
    Throws:
      - applyError on exception

launchPopup

void launchPopup(<string> width, <string> height, <string> wname, <string> trigger, <object> elem)
    Launch popup.
    On the basis of parameter trigger we builds the url string for later use in func window.open. According to the popup launch, the parent window opacity will be lowered to eye focus onto the launched window.

    Example: 
    
Helper.launchPopup('745','634','pages_links_select','pages_internal_links', this);
    Parameters:
      width - Width for the window to launch
      height - Height for the window to launch
      wname - The Name for the window to launch
      trigger - Switch case condition which url to use
      elem - Current element
    Throws:
      - applyError on exception
    See:
      - lowerOpacity()
